# Changelog

## 2.2.0 — 2026-07-13

- Add append-only per-feature workflow events, derived-state explanations, and chronology validation without making events authoritative.
- Persist independent evaluator responses and require their contract/evidence before M10.
- Version and SHA-256 evaluator artifacts so contract upgrades do not invalidate or invite edits to immutable historical reviews.
- Require criterion-level evaluator evidence and core planning sections; support controlled reopen/increment event cycles.
- Enforce controller-only research closeout plus pre-M9 findings, verification, artifact, and `GP-RESEARCH-*` disposition contracts.
- Materialize low-risk batch answers into atomic decision rows with a deterministic CLI command.
- Close lifecycle, readiness, simulated-approval, PDR-reference, and duplicate-research validation gaps found by the v2.1 practice run.

## 2.1.0 — 2026-07-13

- Define canonical workflow states, artifact statuses, M9/M10 semantics, authority modes, and prepared-versus-accepted handoff.
- Replace positional gap-log references with stable atomic `GP-*` decision IDs while retaining low-risk batch question UX.
- Route scenario-hardening changes through small-change, increment, or reopened gap pass before rows may close.
- Add deterministic workflow validation with PowerShell and shell wrappers plus JSON output.
- Add installed-pack manifests, upgrade checks, safe upgrades for templates and managed helper scripts, three-way conflict packets, and explicit acceptance of resolved local content.
- Add evaluator output/retry contracts, simulation mode, portable `markdown-digest` research delivery, and spec-stub ownership validation.

## 2.0.0 — 2026-07-10

- Simplify the core workflow, remove the memory phase, add routing and scenario hardening, and make discovery memory-system-agnostic.
